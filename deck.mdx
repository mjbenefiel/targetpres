<ol style={{ margin: '5px 0', color:'#36CAD9', fontSize:'4vw' }}>What is Adobe Target?</ol>

<ul>
  <Appear>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>Adobe Target is a rule-based testing and targeting tool, which can integrate with Analytics & create reports that can be used for making marketing offers, personalization and UX testing; allowing marketers to figure out which of the offers, experiences & messages are truly engaging the customers.</ol>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>Simply put, it's an A/B testing tool.</ol>
  </Appear>
</ul>

<ol style={{ margin: '5px 0', color:'#36CAD9', fontSize:'4vw' }}>Key Concepts</ol>

<ul>
  <Appear>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>A/B Testing: An activity type that compares two or more versions of your site's content to see which version best improves your conversions during a pre-specified test period.</ol>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>An A/B test (sometimes referred to as an A/B...N test) compares two or more versions of a web site's content to see which best lifts conversions, sales, or other identified metrics. Use an A/B test to compare changes to a page against the default page design to determine which experience produces the best results.</ol>
  </Appear>
</ul>

<ol style={{ margin: '5px 0', color:'#36CAD9', fontSize:'4vw' }}>How Adobe Target Works</ol>

<ul>
  <Appear>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>Adobe Target integrates with websites by means of one of two JavaScript libraries: at.js or mbox.js</ol>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>The at.js library is the new implementation library for Target. The at.js library improves page-load times for web implementations and provides better implementation options for single-page applications. The at.js library also includes the components that were included in target.js, so there is no longer a call to target.js.</ol>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>The mbox.js library is the legacy implementation library for Target. The mbox.js library is still supported, but there will be no feature updates.</ol>
   

  </Appear>
</ul>

<ol style={{ margin: '5px 0', color:'#36CAD9', fontSize:'4vw' }}>How Adobe Target Works (cont'd)</ol>

<ul>
  <Appear>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>Each time a visitor requests a page that has been optimized for Target, a request is sent to the targeting system to determine what content to serve to a visitor. This process occurs in real-time — every time a page is loaded, a request for the content is made and fulfilled by the system. The content is governed by the rules of marketer-controlled activities and experiences and is targeted to the individual site visitor. Content is served so that each site visitor is most likely to respond to, interact with, and ultimately purchase, to maximize response rates, acquisition rates, and revenue.</ol>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>You must reference the Target JavaScript library file on every page on your site.</ol>

  <Appear>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>An mbox is a “marketing box,” which is an area on a web page used by Adobe Target to show different content to visitors in a campaign. Marketers can replace blocks of content in the mbox with dynamic content that they’ve created in Adobe Target.  The content displayed inside an mbox during a campaign is called an offer.</ol>  
      <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>Adobe Target has two methods that can be used to modify content: Place Regional mboxes in pages in the exact locations where content will be modified. Place a Global mbox create statement in the page and define all logic within the Adobe Target interface as to which content will be modified.</ol>  
  </Appear>
</ul>

<ol style={{ margin: '5px 0', color:'#36CAD9', fontSize:'3vw', fontStyle: 'italic' }}>What is Flicker?</ol>

<ul>
  <Appear>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>Flicker is the occurrence of default content being momentarily displayed to the visitor before it is replaced with the content you intended to show the visitor, making it seem to the visitor that the content flickered. Visitors can sometimes interpret flicker as a problem with the site, potentially causing the visitor to lose confidence in the site or brand. Flicker can also cause visitor dissatisfaction if, for example, they see a discount they like "flicker away" to be replaced with a lesser discount. Flicker is a symptom of encountering a delay in loading the content displayed, also called "tests", asynchronously.</ol>  
  </Appear>
</ul>

<ol style={{ margin: '5px 0', color:'#36CAD9', fontSize:'4vw' }}>Adobe Target and TiQ (cont'd)</ol>
<ol style={{ margin: '5px 0', color:'#36CAD9', fontSize:'3vw', fontStyle: 'italic' }}>Flicker Free</ol>

<ul>
  <Appear>
    <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>Tealium supports both asynchronous and synchronous versions of the Adobe Target tag. The asynchronous version is recommended. However, if you choose to load your tests synchronously, you will not see flicker occur. However, loading content synchronously means that the entire page must wait to receive that content before it parses and displays the rest of the page.</ol>  
      <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>Asynchronous loading circumvents the issue of unavailable content halting page load because the browser continues to load the rest of the page while it waits for the pending content. However, if there is a long enough delay between the browser encountering the offer on the page and retrieving the offer's content, visitors may briefly see the default content. This is the flicker phenomenon.</ol>  
  </Appear>
</ul>

The content is not removed, but the browser will not render it. The layout dimensions of the hidden content are preserved, so the layout of your page is not deformed. To a visitor it will appear as though the section where the content would be displayed is blank instead. There is no flicker.</ol>  
     <ol style={{ margin: '20px 0', color:'#959BA1', fontSize:'2vw' }}>When the utag.js file loads, it calls Adobe Target which in turn shows the hidden nodes.</ol> 
     
  </Appear>
